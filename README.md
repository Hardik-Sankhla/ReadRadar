## The Problem

The AI ecosystem is exploding.

Every day:
- New papers are published
- New models are released
- New repositories appear
- New courses launch

Engineers waste hundreds of hours trying to answer:
*"What should I learn next?"*

ReadRadar exists to solve that problem.

## What ReadRadar Does

ReadRadar transforms fragmented technical resources into structured learning intelligence.

Instead of showing thousands of disconnected links, ReadRadar provides:
- Curated Learning Paths
- Resource Recommendations
- Trust Indicators
- Knowledge Relationships
- Topic Collections
- Progressive Learning Journeys

## Architecture

ReadRadar is the frontend viewer for an intelligence pipeline. It relies on a decoupled, modular architecture:

1. **BookOS** (The Workers): A suite of autonomous Python agents that discover, validate, and enrich metadata across the internet (GitHub, ArXiv, Hugging Face, Publishers).
2. **KnowledgeRegistry** (The Truth): A standalone repository that acts as a Git-backed NoSQL database, storing the rich JSON structures and graphs generated by BookOS.
3. **ReadRadar** (The Viewer): This repository. It consumes the `KnowledgeRegistry` and renders them into the neo-brutalist intelligence dashboard you see. ReadRadar is completely ignorant of how the data is generated, making the frontend lightning fast, scalable, and independent.
